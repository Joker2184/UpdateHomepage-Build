超大的底层更改！
……但由于是底层更改，所以虽然做了老久，但大家用起来不一定有什么不同就是了 Orz

## 主要调整
- 将运行所需的环境从 .NET Framework 4.6.2 调整为 .NET Framework 4.8
  - 如果你的 .NET Framework 版本过低，打开 PCL 时会自动提示下载
- 新主页预设：整合包推荐引擎（#8588，作者 @CHMYGZ）
- 使用 Modrinth 进行搜索时，支持筛选 LiteLoader
- 将 Minecraft 崩溃弹窗中的“查看日志”改为“查看输出”，点击会提供游戏的原始输出信息，以便于分析在 JVM 初始化阶段就发生的崩溃
- 若开启调试模式，会将 Minecraft 原始输出也输出到 PCL 日志文件，以便进行错误分析

## 次要调整
- 在指向按钮等可点击对象时，鼠标光标改为手形（#7048，@0x1A5，PR @wyc-26）
- 只在 AccessToken 过期时刷新微软登录，以避免重复打开 PCL 时触发登录太过频繁的问题（#8018，@Qulinlins，PR @0x1A5）
- 未在组策略开启长路径兼容时，现在也兼容使用长路径（#8446，@Howard302）
- 仅为 LWJGL 3.4.1 启用 LWJGL Unsafe Agent（#8464，PR @Big-Cake-jpg）
- 修改兼容的 TLS 版本，提高安全性（#8519，@MatchaOre）
- 仅在非 GBK 编码、非 ASCII 文件夹下、未自定义 javaagent 时，才启用 Java Launch Wrapper，以优化部分 Mod 的兼容性（#8539，@DeliciousBread481）
- 移除主页预设：简单主页、PCL 主页市场
  - 简单主页将在后续调整托管平台，然后重新加入
- NetRequestByClientRetry 不再对 (404) NotFound 进行重试
- 搜索、导入 Java 的按钮改为 MyIconTextButton，以与其他按钮的样式统一
- 在 PCL 没有文件夹权限时，会给出更详细的错误信息
- 检查 assets 和 libraries 文件夹下的文件是否可以复用时，会对文件大小进行缓存

## API 更改
- XAML 中，允许用 core 和 corewpf 命名空间来调用 MeloongCore 中的类
- XAML 中，将用于绑定的自定义 Converter 从 local 命名空间迁移到 corewpf 命名空间
- 修复了自定义 JVM 与游戏参数无法使用 PCL 替换标记的 Bug

## 技术性更改
- 将 MeloongCore 的框架升级到 .NET Standard 2.0（Meloong-Git/MeloongCore#2，@LinQingYuu）
- 重构日志系统
  - 支持指定日志等级
    - 现在分为 Trace、Info、Warn、Error 四个等级
    - If ModeDebug Then Log(...) 现在是 Logger.Trace(...)
    - Trace 级别的日志只在开启调试模式时，或在 Debug 构建中，才会输出
  - 日志会自动记录调用方的文件名，而非手动指定模块名
  - 在写入多行日志时，每行都会有时间戳等日志头了，而不是只在第一行才有
  - 日志写入现在是线程安全的
  - 将日志写入到调试输出的过程将在另一线程执行，这能大大提高在 VS 调试时的性能
  - Trace 级别的日志若不会被写入，则不会尝试生成对应的内插字符串，以提高性能（此优化仅对 C# 代码有效）
  - 核心代码已迁移到 MeloongCore 的 Logger，部分迁移到 PclLogger
- 重构所有 IO 操作：文件、文件夹、压缩、解压、Stream 等
  - 所有 IO 操作都会使用 \\?\ 扩展路径格式，以更好地支持长路径
  - 核心代码已迁移到 MeloongCore 的 FileUtils 与 DirectoryUtils
- 重构大量其他基础库代码
- 添加 MeloongCore.Test 测试项目
- 将 Settings.Get 改为 Settings.Get(Of T)，以确保类型安全
- 修改 Mod 数据库格式，以减少未来的增量补丁的大小
- 移除 EqualableList 类
- 不再使用 VB 的“将函数名作为返回变量”特性
