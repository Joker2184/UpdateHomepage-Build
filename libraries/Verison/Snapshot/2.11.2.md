## PCL 联机功能（再次）暂时关闭的说明……

很悲伤的是，联机功能确实遇到了亿些麻烦……

- 各个节点连续遭受了一周多不明来源的 DDoS 攻击，所以经常有玩着玩着掉线，或者找不到节点的情况。最近这两天更凶残，统计到的联机失败率都干到 50% 了……我们计划换一个不对外公开节点信息的负载均衡方案，来缓解所有节点被集体 DDoS 的问题。

- 目前的模式在合规性上有着一些问题。为了避免给 EasyTier 社区，以及各个节点的提供者带来麻烦，我们计划在完全解决合规问题后再开放共享节点搭建，这样就会稳很多！

- 最近两天有一些商家在有组织地带节奏，呃，封面和标题基本都一样的。在联机功能本来就不稳定的当下，我们难以估摸他们会不会做出更过激的事情……

种种麻烦加起来，我们实在是没有灵丹妙药，能在短期内嘎嘎解决这么一大堆问题。所以在我们挨个解决这些问题前，只能先暂时关闭 PCL 的联机功能……

<paracolor color="Orange"/>在和 EasyTier 开发者寻思之后，我们觉得我们还是有希望解决上面的这些问题，打赢复活赛的！未来，我们或许能够端上来一个更加完善、合规、抗攻击的架构，尝试解决前面的这些麻烦问题，并且为大家继续免费地开放联机！

## 联机优化

- 若有加入者的网络环境比房主更好，会提示可以让那位加入者担任房主
- 中继节点负载均衡算法优化
- 只选择 MC 中继节点进行连接，避免连接到低质量节点导致卡顿
- 大幅降低节点在线率在负载算法中的权重
- 允许自动选择非 TCP 协议的、带有即将下线标签的、在线状态不是优秀的节点
- 会有 1% 的概率选择 0 负载的节点，而不是固定不选择它们（如果节点存在问题，可能才导致它是 0 负载，所以此前不会选择它们，但这又会导致 PCL 不会连接新增的节点）
- 现在会在多个禁止中继的节点中选择一个作为发现节点，以实现负载均衡
- 会检查是否与房主都设置了自定义节点，以防只有一方设置了导致连不上
- 允许设置优先直连或优先低延迟，以降低节点中转负载
- 为侦听的端口号添加了一定的随机性，以降低端口被占用的概率（[#7451](https://github.com/Meloong-Git/PCL/issues/7451)，@dongtz123）
- 若正使用中继节点，会更准确地显示网络情况
- 仅创建房间时不再连接中继节点
- 为节点列表 API 添加了缓存作为回退，以提高可用性
- 联机设置的贡献节点选项会给出更详细的说明
- 若联机模块因为端口被占用而崩溃，会给出详细的解决方案
- <paracolor color="Orange"/>更新联机的内置帮助，现在离线模式直接选择房间就能加入，不应手动输入 IP（Help[#437](https://github.com/Meloong-Git/PCL/issues/437)，PR @kuankqaq）
- 修复了以下 4 个联机相关的 Bug
  - 新用户首次启动联机可能出错（[#7346](https://github.com/Meloong-Git/PCL/issues/7346)）
  - <paracolor color="Orange"/>俄语等环境下无法正常联机（[#7435](https://github.com/Meloong-Git/PCL/issues/7435)，@Arrogant-BOT）
  - 创建房间时会显示“正在连接到房主”
  - 启动联机时会冗余地检查一次配置
